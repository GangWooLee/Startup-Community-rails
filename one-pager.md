# **1️⃣ Product Vision (제품 비전)**

“아이디어·사람·외주가 한 공간에서 연결되는 최초의 창업 커뮤니티.”

한국 초보 창업자는 정보도 부족하고, 외주를 맡길 곳도 흩어져 있으며, 네트워크도 없다.

우리는 **커뮤니티 기반의 신뢰 데이터**와 **외주 기능**을 결합하여

“처음 창업하는 사람들이 시행착오 없이 시작하도록 돕는” 플랫폼을 만든다.

---

# **2️⃣ Problem Statement (문제 정의)**

### **❗ 한국의 초기 창업자들은 다음 문제를 겪고 있음**

1. **커뮤니티는 있으나 외주 정보와 연결되지 않는다.**  
2. **크몽/위시켓은 외주만 있고 커뮤니티 성격이 없다.**  
3. **아이디어에 대한 피드백을 받을 기회가 부족하다.**  
4. **신뢰 기반 프로필이 없어 사람을 구하기 어렵다.**  
5. **정보가 파편화되어 있다 (에브리타임/인스타/카카오톡/구글폼).**

---

# **3️⃣ Goals (제품 목표)**

### **🎯 1차 MVP의 가장 중요한 목표**

“커뮤니티 활동 → 프로필 → 외주 공고”가 자연스럽게 연결되는 사용자 흐름 구축

### **구체적 목표**

* 사용자가 글을 올릴 수 있고, 대화를 나눌 수 있다.  
* 글쓴이의 프로필에서 **그 사람의 외주 구인/구직 게시물**을 확인할 수 있다.  
* 외주 공고 페이지는 최소 정보만 포함해도 클릭해볼 수 있다.  
* 마이페이지에서 **내 활동(스크랩/프로필 수정)** 관리를 할 수 있다.

---

# **4️⃣ Target Users (타깃 사용자)**

* **대학생/초기 창업자**  
* **사이드프로젝트를 하고 싶은 개발자/디자이너/기획자**  
* **외주를 맡기고 싶은 소규모 창업자**  
* **창업 관련 인사이트를 얻고 싶은 일반 사용자**

---

# **5️⃣ User Stories (사용자 스토리)**

### **🟦 Community User**

* “창업 관련 질문을 올리고 피드백을 받고 싶다.”  
* “마음에 드는 사람의 글을 보고 프로필을 보고 싶다.”  
* “누가 어떤 프로젝트를 구하는지 알고 싶다.”

### **🟧 Outsourcing Client (구인자)**

* “프로필을 기반으로 사람이 신뢰할 만한지 알고 싶다.”  
* “내가 올린 구인 글을 한 곳에서 확인하고 싶다.”

### **🟩 Freelancer / Talent (구직자)**

* “내가 제공할 수 있는 서비스(구직)를 정리해 올리고 싶다.”  
* “내 포트폴리오처럼 프로필 페이지에 정리되었으면 좋겠다.”

### **🟨 General User**

* “내가 스크랩해둔 게시물을 한 번에 보고 싶다.”

---

# **6️⃣ Core Features (핵심 기능)**

## **✔️ (1) Community 게시판**

**기능 목표:**

창업자/개발자/디자이너가 양질의 대화를 나누는 공간을 만들고, 프로필로 연결되는 첫 접점을 제공한다.

**주요 기능:**

* 글쓰기  
* 댓글  
* 좋아요  
* 스크랩  
* 공유 (선택)  
* 글쓴이 프로필 클릭 기능

**UX Requirements:**

* 게시글 카드에는 반드시 “작성자 프로필 썸네일 \+ 이름 \+ 기본 역할(직무)” 포함  
* 카드 클릭 시 상세 페이지로 이동  
* 프로필 클릭 시 프로필 페이지로 이동

---

## **✔️ (2) Profile 페이지 (구인/구직 통합 프로필)**

**기능 목표:**

커뮤니티 활동 ↔ 외주 기능을 연결하는 핵심 페이지.

“그 사람은 어떤 사람이고, 어떤 일을 구하거나 제공하는지” 한눈에 보여주는 구조.

**구성:**

* 프로필 정보  
  1. 프로필 사진  
  2. 이름 / 역할 (Founder, Developer 등)  
  3. 간단한 소개  
* 탭 3개  
  1. **Posts** – 작성한 커뮤니티 글  
  2. **Job Posts (구인)** – 외주 요청 글  
  3. **Talent Listings (구직)** – 제공 가능한 서비스  
* 탭 클릭 시 해당 리스트 노출  
* 외주 글 클릭하면 외주 상세 페이지로 이동

**UX Requirements:**

* 카드 디자인은 모두 동일한 규격  
* 리스트는 무한스크롤 or 페이지네이션

---

## **✔️ (3) Outsourcing 공고 페이지 (구인/구직)**

**기능 목표:**

최소 기능으로 외주 정보 탐색을 가능하게 한다.

**포함 요소:**

* 제목  
* 카테고리 (개발/디자인/PM/마케팅)  
* 프로젝트 타입 (단기/장기/일회성)  
* 설명  
* 예산 (optional)  
* 작성자 프로필  
* 지원(문의) 버튼 (1차 MVP에서는 기능 없이 UI만 존재 가능)

**UX Requirements:**

* 상단에 작성자 미니 프로필 표시  
* 버튼 위치는 항상 우측 or 하단 고정  
* 복잡한 기능 없음

---

## **✔️ (4) My Page (마이페이지)**

**기능 목표:**

프로필 관리와 내가 저장한 정보 접근 제공.

**구성:**

* 프로필 수정  
* 내가 스크랩한 글  
* 내가 올린 구인·구직 글  
* 로그아웃

---

# **7️⃣ Functional Requirements (기능 요구사항)**

### **🔵 Community**

* R1: 게시글 CRUD  
* R2: 댓글 CRUD  
* R3: 좋아요 기능  
* R4: 스크랩 기능  
* R5: 작성자 프로필 링크 기능

### **🔴 Profile**

* R6: 프로필 정보 조회/수정  
* R7: 사용자별 게시글 목록  
* R8: 사용자별 구인 목록  
* R9: 사용자별 구직 목록

### **🟣 Outsourcing**

* R10: 구인/구직 공고 생성  
* R11: 공고 상세조회  
* R12: 공고 작성자 프로필 링크

### **🟢 My Page**

* R13: 프로필 편집  
* R14: 내가 스크랩한 글 리스트 보여주기

# **9️⃣ Non-functional Requirements (NFR)**

* **초기 로딩 속도 최소화** (간단한 DB 구조)  
* **반응형 디자인 (모바일 우선)**  
* **초기엔 로그인 없이도 내용 볼 수 있도록 허용 가능**  
* **회원가입은 최소정보(이름·역할·학과 또는 직무·한줄소개)**
